# Grid Analyzer v0 — 技术方案

## 1. 项目概述

| 项目 | 内容 |
|---|---|
| 项目名称 | Grid Analyzer（电网分析仪） |
| MCU | STM32G474RET6 (ARM Cortex-M4, 170MHz) |
| ADC | 2× ADS8685 (16-bit, SPI, Daisy Chain) |
| 隔离芯片 | ISO7741（LV-HV 电气隔离） |
| 通信接口 | LPUART1 (115200 8N1) |
| 状态指示 | LED1 (PD2) + LED2 (PC13) |

## 2. MCU 任务

MCU **不进行 FFT 计算**，仅负责：

1. 通过 SPI 控制两个 ADS8685（daisy chain 连接）进行同步采样
2. 将原始采样数据通过 UART 传输到上位机
3. 通过 LED 指示工作状态
4. 接收并解析上位机的控制指令

FFT 谐波分析由上位机软件完成。

## 3. 采样参数

| 参数 | 值 | 说明 |
|---|---|---|
| 采样率 | **32768 SPS** | 定时器 TIM 触发 ADC CONVST |
| 采样点数 | **16384 点/通道** | 电压 + 电流 双通道 |
| 采集时长 | 16384 / 32768 = **0.5 秒** | 覆盖 25 个 50Hz 周期 |
| 频率分辨率 | 32768 / 16384 = **2 Hz** | 可分辨基波(50Hz)至 50 次谐波(2500Hz) |
| 数据总量 | 16384 × 2 × 2 = **65536 bytes (64KB)** | 电压 32768B + 电流 32768B（仅存低 16-bit 有效数据） |
| MCU SRAM | STM32G474 128KB SRAM | 64KB 数据存放无压力 |

## 4. 硬件连接

参考硬件架构图 `grid analyzer architecture ADS8685.svg`。

### SPI Daisy Chain 连接

```
STM32 SPI3_MOSI ──→ ISO7741 ──→ ADS8685_1.SDI ──→ ADS8685_1.SDO ──→ ADS8685_2.SDI
STM32 SPI3_MISO ←── ISO7741 ←── ADS8685_2.SDO
STM32 SPI3_SCLK ──→ ISO7741 ──→ ADS8685_1.SCLK & ADS8685_2.SCLK
STM32 GPIO PA15 ──→ ISO7741 ──→ ADS8685_1.CONVST/nCS & ADS8685_2.CONVST/nCS
```

- SPI 配置：Master, 16-bit, CPOL=0, **CPHA=0** (SPI-00-S), NSS=Software, Baud = 10.625MHz
- 与 ADS8685 上电默认 SPI-00-S 协议一致，无需额外配置协议寄存器
- CONVST/CS 引脚：**PA15**（GPIOA, OUTPUT_PP, 初始高电平），作为 SPI 的软件 nCS 和 ADS8685 的 CONVST 合二为一
- ADS8685 Daisy Chain 要求：每帧 SCLK 边沿数 = **32 × N**（N=2 → 64 个边沿）
- SPI3 为 16-bit data size → 每次 daisy chain 读取需要 **4 × 16-bit 连续传输**（共 64 SCLK 边沿）

### UART 连接

```
STM32 LPUART1_TX ──→ 上位机 RX
STM32 LPUART1_RX ←── 上位机 TX
```

### 6.2 状态切换条件

| 转换 | 条件 | 触发事件 | 说明 |
|---|---|---|---|
| IDLE → ADC_CONV | **A** | 上位机发送 `START` 指令 | 收到完整校验正确的指令帧 |
| ADC_CONV → DATA_TRANS | **B** | 采集满 16384 点 | sample_count 计数达到最大值 |
| DATA_TRANS → IDLE | **C** | UART 数据帧发送完成 | TX 完成中断回调 |

### 6.3 同步机制（中断 + 主循环轮询）

```
UART RX 中断 ──→ 收到 START 指令 ──→ 置位 evt_start_received
TIM 中断(32768Hz) ──→ 触发 ADC CONVST + 读取数据
                   ──→ sample_count++ ──→ 满16384时置位 evt_sample_done
UART TX 完成中断 ──→ 置位 evt_tx_done

main loop (轮询):
  switch(state) {
    case IDLE:     if (evt_start_received)  state = ADC_CONV;   break;
    case ADC_CONV: if (evt_sample_done)     state = DATA_TRANS; break;
    case DATA_TRANS: if (evt_tx_done)       state = IDLE;       break;
  }
```

### 6.5 异常处理

| 异常场景 | 处理方式 |
|---|---|
| ADC_CONV 中收到新 START | 忽略，继续当前采集 |
| DATA_TRANS 中收到新 START | 忽略，完成传输后回到 IDLE 再处理 |
| SPI 读取超时/失败 | 丢弃本次采集数据，返回 IDLE，置错误标志 |

## 9. 数据流

```
上位机 ──START指令──→ STM32 (UART RX ISR置位事件)
         ↓
    进入 ADC_CONV：CONVST=0 初始化，启动 TIM2
         ↓
    TIM2 ISR 触发 32768Hz (30.5µs周期) —— 以下为一个采样点
         ↓
    ① CONVST↑（上升沿 → 采样输入 + 启动转换，SDO→tri-state）
    ② delay 5µs（t_CONV ≈ 4µs）
    ③ CONVST↓（下降沿 → 结果就绪 + SDO 驱动）
    ④ delay 1µs（t_CSSD，等待 SDO 稳定）
    ⑤ SPI 4×16-bit 全双工读取（64 SCLK 边沿）
         ↓
    SPI 读取结果：先收到 ADC2 低16b(电流)，后收到 ADC1 低16b(电压)
         ↓
    存入 RAM 缓冲区 [current_buf[sample_count], voltage_buf[sample_count]]
    CONVST 保持 0（等待下一次 ISR）
         ↓
    sample_count++ → 到 16384 → 停止 TIM2
         ↓
    组数据包（帧头 + 电压数据 + 电流数据 + 校验和）
         ↓
    UART 阻塞发送 (65569 bytes, ~5.7s @115200)
         ↓
    发送完成中断 → 返回 IDLE
```

## 10. ADS8685 Daisy Chain 采样方法

### 10.1 硬件拓扑

```
MCU SPI3_MOSI ──→ ISO7741 ──→ ADC1.SDI
MCU SPI3_MISO ←── ISO7741 ←── ADC2.SDO
MCU SPI3_SCLK ──→ ISO7741 ──→ ADC1.SCLK ──→ ADC2.SCLK
MCU GPIO PA15 ──→ ISO7741 ──→ ADC1.CONVST/nCS ──→ ADC2.CONVST/nCS
ADC1.SDO ──→ ADC2.SDI   ←-- Daisy Chain 级联
```

两个 ADC 共享 SCLK 和 CONVST/nCS，ADC1 的 SDO 连接到 ADC2 的 SDI 实现数据级联。

### 10.2 每个采样数据的 bit 数

| 项目 | 数值 |
|---|---|
| ADS8685 ADC 分辨率 | **16-bit** |
| 每通道每个采样点数据 | **16-bit**（1 word） |
| Daisy chain 一次传输总 bit 数 | **64** SCLK 边沿 = 32 × N × 1-edge = 64-bit |
| 一次传输中每个器件的帧 | **32-bit** / 器件（高 16-bit 状态 + 低 16-bit 有效数据） |
| 一次传输提取的有效数据 | ADC1 低 16-bit（电压）+ ADC2 低 16-bit（电流）= 32-bit |

### 10.3 采样流程

#### 步骤 1 — 启动转换（同一 ISR 内完成）
每个 TIM2 中断内完成一次完整的"采样 + 转换 + 读取"：

1. **CONVST↑（上升沿）** → 两个 ADC 同时采样输入信号，启动转换（SDO 变 tri-state）
2. **等待 5µs（t_CONV ≈ 4µs）**，确保转换正常开始
3. **CONVST↓（下降沿）** → 转换结果加载到输出移位寄存器，SDO 被驱动
4. **等待 1µs（t_CSSD）**，等待 SDO 输出稳定
5. **SPI 传输 64 个 SCLK 边沿**，完成 4 次 16-bit 读取（全双工）
6. CONVST 保持低电平，等待下一次 ISR 触发

> **关键**：同一 ISR 中 CONVST↑ 触发的是"本次"采样点的转换，随后立即读取。buffer[N] 存储的就是第 N 个采样点的真实数据，无流水线延时。

#### 步骤 2 — SPI 读取（16-bit 模式，四拍完成）

Daisy chain 要求 **32 × N SCLK 捕获边沿**（N=2 → 64 边沿）。SPI3 配置为 16-bit data size（16 SCLK/传输），需要 **4 次连续 16-bit SPI 传输**。数据在 CONVST/CS 为低电平时传输，CONVST/CS 上升沿启动转换。

**数据接收顺序**（依据手册：先收到 Device N，再收到 Device N-1）：
- 链尾 ADC2 是 Device N → **先被接收**
- 链首 ADC1 是 Device N-1 → **后被接收**

| 传输序号 | SCLK 边沿 | MOSI 发送 | MISO 接收 | 数据内容 |
|---|---|---|---|---|
| **第 1 次** | 1-16 | dummy 16-bit | 16-bit | ADC2 帧**低 16-bit**（**电流有效数据**） |
| **第 2 次** | 17-32 | dummy 16-bit | 16-bit | ADC2 帧**高 16-bit**（状态位，丢弃） |
| **第 3 次** | 33-48 | dummy 16-bit | 16-bit | ADC1 帧**低 16-bit**（**电压有效数据**） |
| **第 4 次** | 49-64 | dummy 16-bit | 16-bit | ADC1 帧**高 16-bit**（状态位，丢弃） |

#### SPI 读取时序详解

```
CONVST/CS ───────\____________________________________________/↑─── 采样边沿
                   ↓ 数据帧开始         ↓          ↓          ↓
SCLK  ──_─_─_─ 16-cycle ──_─_─_─ 16-cycle ──_─_─_─ 16-cycle ──_─_─_─ 16-cycle
                   ↑          ↑          ↑          ↑
MOSI  ─X dummy1 X───X dummy2 X───X dummy3 X───X dummy4 X──
MISO  ─X ADC2低16bX───X ADC2高16bX───X ADC1低16bX───X ADC1高16bX──
            ↓               (丢弃)           ↓               (丢弃)
        电流有效数据                        电压有效数据
```

#### 数据提取说明

每个 ADC 的 32-bit 帧中：
- **高 16-bit（[31:16]）**：配置状态/寄存器的回读数据，**可丢弃**
- **低 16-bit（[15:0]）**：**ADC 转换有效数据**，存入缓冲区

### 10.4 读取顺序总结

| 传输序号 | 数据来源 | 低 16-bit 含义 |
|---|---|---|
| **第 1 次** | ADC2 帧低 16-bit | **电流通道** 采样有效值 (16-bit) |
| **第 2 次** | ADC2 帧高 16-bit | ADC2 状态（丢弃） |
| **第 3 次** | ADC1 帧低 16-bit | **电压通道** 采样有效值 (16-bit) |
| **第 4 次** | ADC1 帧高 16-bit | ADC1 状态（丢弃） |

### 10.5 TIM2 ISR 实现 (采样+转换+读取在同一个中断)

CONVST 在 ISR 间保持低电平。每次 ISR 触发时：先上升沿启动转换 → 延时等待 → 下降沿读取结果：

```c
// TIM2 中断回调 — 每 30.5µs 触发一次 (32768 Hz)
void HAL_TIM_PeriodElapsedCallback(TIM_HandleTypeDef *htim)
{
    if (htim->Instance == TIM2) {
        adc_ads8685_read_sample();
    }
    else if (htim->Instance == TIM3) {
        led_indicator_tick();
    }
}
```

`adc_ads8685_read_sample()` 的实现：

```c
void adc_ads8685_read_sample(void)
{
    uint16_t tx_buf[4] = {0, 0, 0, 0};  // 4×16-bit = 64 SCLK
    uint16_t rx_buf[4];

    // ① CONVST↑（上升沿 → 采样 + 启动转换，SDO 变 tri-state）
    HAL_GPIO_WritePin(CONVST_GPIO_Port, CONVST_Pin, GPIO_PIN_SET);

    // ② 等待 5µs（t_CONV ≈ 4µs）
    delay_us(5);

    // ③ CONVST↓（下降沿 → 结果就绪 + SDO 被驱动）
    HAL_GPIO_WritePin(CONVST_GPIO_Port, CONVST_Pin, GPIO_PIN_RESET);

    // ④ 等待 1µs（t_CSSD），等待 SDO 稳定
    delay_us(1);

    // ⑤ 全双工 SPI 读取 4×16-bit（共 64 SCLK）
    //    发送 dummy 数据，同时读回 MISO
    HAL_SPI_TransmitReceive(&hspi3, (uint8_t*)tx_buf, (uint8_t*)rx_buf, 4, HAL_MAX_DELAY);

    // 接收顺序：先 ADC2（链尾），后 ADC1（链首）
    // 每个 32-bit 帧中：先低 16-bit（有效数据），后高 16-bit（丢弃）
    adc_current_buf[sample_count] = rx_buf[0];  // ADC2 低 16b → 电流
    adc_voltage_buf[sample_count] = rx_buf[2];  // ADC1 低 16b → 电压
    sample_count++;

    // CONVST 保持 LOW，等待下一次 ISR

    if (sample_count >= 16384) {
        sample_count = 0;
        evt_sample_done = 1;
        HAL_TIM_Base_Stop_IT(&htim2);
    }
}
```

> **关键点**：
> - buffer[N] 存储的是第 N 个采样点的真实数据，**无流水线延时**
> - ISR 总耗时 ≈ 5µs + 1µs + 6µs ≈ **12µs**，远小于采样周期 30.5µs
> - 进入 ADC_CONV 时需初始化 CONVST=0，再启动 TIM2

### 10.6 SPI 配置参数

| 参数 | 值 | 说明 |
|---|---|---|
| Mode | Master | MCU 为主机 |
| Data Size | 16-bit | ADS8685 为 16-bit 器件 |
| CPOL | Low (0) | SCLK 空闲为低 |
| CPHA | **1 Edge (0)** | SPI-00-S：上升沿捕获，下降沿移位 |
| NSS | Software | 软件控制，通过 PA15 手动操作 nCS |
| Baud Rate | Prescaler = 16 → **10.625 MHz** | 170MHz / 16 |
| First Bit | MSB First | 高位先行 |

### 10.7 ADS8685 初始化配置

根据 ADS8685 Datasheet (SBAS633E) Section 7.5.2 和 7.6：

**1. 设置输入范围 ±2.56V (RANGE_SEL_REG, 地址 14h)**

WRITE 命令格式：`11010_00_<9-bit address>_<16-bit data>`
- 9-bit address = 0_0001_0100 = 0x014
- RANGE_SEL[3:0] = 0100b (±0.625 × VREF = ±2.56V)
- **命令字 = 0xD0140004**

**2. Daisy Chain 配置要点**

不需要额外设置 "Daisy Chain 使能" 寄存器——ADS8685 在 SDO_CTL_REG[7:0] = 00h 时，长数据帧（SCLK > 32）自动进入 daisy chain 模式（见 Section 7.5.4.2.1）。

只需要确保 SDO_CTL_REG (0Ch) 保持上电默认值 0x0000 即可。如果需要明确写入：
- **命令字 = 0xD00C0000**（写默认值）

**3. 初始化顺序**

每个配置操作需通过菊花链发送 **64-bit 帧**（4×16-bit），前 32-bit 到达远端芯片（ADC1），后 32-bit 到达近端芯片（ADC2）。

```
上电等待 → 拉低 CONVST
  → SPI 64-bit 写 {0xD0140004, 0xD0140004} → 拉高 CONVST (两个芯片同时配量程 ±2.56V)
  → 拉低 CONVST
  → SPI 64-bit 写 {0xD00C0000, 0xD00C0000} → 拉高 CONVST (确认 SDO_CTL)
  → 拉低 CONVST (进入采集状态)
```

在 daisy chain 模式下，两个 ADC 共享相同的 CONVST/CS，**一次 64-bit 传输同时配置两个芯片**。

> 初始化完成后，两个芯片即可在每 30.5µs 的 TIM2 中断中同步采样。

## 11. 性能估算

| 操作 | 耗时 | 说明 |
|---|---|---|
| 单点采样+SPI读取 | < 10 µs | SPI 10.625MHz, 64-bit ≈ 6µs |
